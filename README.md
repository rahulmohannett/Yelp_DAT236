## Installation & Setup

### Step 1: Clone the Repository

```bash
git clone <repository-url>
cd Yelp_prototype
```

### Step 2: Database Setup

Start MySQL and run the initialization script:

```bash
# Log in to MySQL
mysql -u root -p

# From inside MySQL shell:
source database/init.sql;
exit;
```

This creates the `yelp_db` database with all tables and inserts sample data (2 test users, 6 restaurants, 5 reviews, 1 user preference record).

To verify:

```bash
mysql -u root -p -e "USE yelp_db; SHOW TABLES; SELECT COUNT(*) FROM restaurants;"
```

You should see 7 tables and 6 restaurants.

### Step 3: Backend Setup

```bash
cd backend

# Create and activate virtual environment
python3 -m venv venv
source venv/bin/activate        # macOS/Linux
# venv\Scripts\activate          # Windows

# Install dependencies
pip install -r requirements.txt

# Create environment file
cp .env.example .env
```

Now edit `backend/.env` with your actual credentials:

```
DATABASE_URL=mysql+pymysql://root:YOUR_MYSQL_PASSWORD@localhost:3306/yelp_db
SECRET_KEY=any-random-secret-string-here
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
GOOGLE_API_KEY=your-actual-google-gemini-api-key
TAVILY_API_KEY=your-actual-tavily-api-key
CORS_ORIGINS=http://localhost:5173,http://localhost:3000
```

### Step 4: Frontend Setup

```bash
cd ../frontend    # from backend directory

# Install dependencies
npm install

# Create environment file
cp .env.example .env
```

The default `frontend/.env` should contain:

```
VITE_API_URL=http://localhost:8000
```

## Running the Application

You need **two terminal windows** — one for backend, one for frontend.

### Terminal 1: Start Backend

```bash
cd backend
source venv/bin/activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Verify by visiting: http://localhost:8000 — you should see `{"message":"Welcome to Yelp Prototype API","docs":"/docs","version":"1.0.0"}`

### Terminal 2: Start Frontend

```bash
cd frontend
npm run dev
```

Verify by visiting: http://localhost:5173 — you should see the home page with "Discover Your Next Favorite Restaurant."

### Quick Verification Checklist

| Check                         | URL                              | Expected                          |
|-------------------------------|----------------------------------|-----------------------------------|

| Health check                  | http://localhost:8000/health     | `{"status":"healthy"}`            |
| Frontend running              | http://localhost:5173            | Home page with chatbot prompt     |

## API Documentation (Swagger)

FastAPI auto-generates interactive Swagger documentation at:

- **Swagger UI:** http://localhost:8000/docs — click "Try it out" on any endpoint
- **ReDoc:** http://localhost:8000/redoc — alternative read-only docs

To authenticate in Swagger:

1. Call `POST /auth/login` with `{"email":"customer@test.com","password":"password"}`
2. Copy the `access_token` from the response
3. Click the **Authorize** button (top right of Swagger UI)
4. Enter: `Bearer <your_token>` and click Authorize
5. All protected endpoints will now work

## Troubleshooting

**"Module not found" when starting backend:**
Ensure virtual environment is activated (`source venv/bin/activate`) and dependencies are installed (`pip install -r requirements.txt`).

**Database connection error:**
- Verify MySQL is running: `mysqladmin ping -u root -p`
- Verify `DATABASE_URL` in `.env` matches your MySQL credentials
- Verify the database exists: `mysql -u root -p -e "SHOW DATABASES;" | grep yelp_db`

**CORS errors in browser console:**
- Ensure backend is running on port 8000
- Ensure `CORS_ORIGINS` in backend `.env` includes `http://localhost:5173`

**AI chatbot returns errors:**
- Verify `GOOGLE_API_KEY` is valid and has Gemini API access
- Verify `TAVILY_API_KEY` is valid
- Check backend terminal for detailed error logs

**Frontend shows blank page:**
- Check browser console for errors
- Ensure `VITE_API_URL` in `frontend/.env` is `http://localhost:8000`
- Ensure backend is running

**"401 Unauthorized" on protected endpoints:**
- Token may have expired (default 30 min). Login again.
- Ensure the Authorization header is `Bearer <token>` (with space after Bearer).

**Cannot claim restaurant (400 error):**
- Restaurant may already have an owner. Only unclaimed restaurants (owner_id = NULL) can be claimed.
- Unclaimed sample restaurants: Taco Fiesta (3), Green Leaf Cafe (4), The Golden Wok (5), Bombay Spice (6).
